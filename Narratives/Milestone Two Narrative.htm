Milestone Two Narrative: Artifact 1 (Software Engineering & Design)

Artifact Description

The first artifact I selected for enhancement is the Reverse‑Engineered C++ Client Management Program originally created in CS‑410. This artifact was developed as part of a reverse‑engineering proficiency test, where I reconstructed a menu‑driven client management system from a compiled binary. The original program included a simple text‑based interface with options to change customer choice, display customer information, or exit. It relied heavily on global state, contained placeholder helper routines, and used hardcoded values throughout.

This artifact was created in 2024 and represents one of my earliest demonstrations of software engineering fundamentals, including program structure, user input handling, and basic system reconstruction from assembly‑level analysis.

Justification for Inclusion in My ePortfolio

I selected this artifact because it provides a clear opportunity to demonstrate my growth in software engineering, modular design, secure coding, and maintainability. In my Milestone One code review, I identified several issues with the original implementation:

  • Overreliance on a global mutable state variable
  • No abstraction or separation of concerns
  • Hardcoded values, including sensitive placeholder data
  • Minimal input validation and no exception handling
  • Tightly coupled menu logic
  • No documentation or architectural clarity

These weaknesses made the artifact an ideal candidate for enhancement, allowing me to showcase my ability to transform a monolithic, insecure program into a clean, modular, and professional‑quality software component.

The enhanced version demonstrates:

  • Object‑oriented design through the introduction of MenuController, CustomerInfo, and PermissionManager classes
  • Secure coding practices, including input validation, safe state transitions, and removal of hardcoded PII
  • Improved maintainability through configuration constants, clear method responsibilities, and structured control flow
  • Professional documentation and readable, well‑organized code

This artifact now reflects the level of engineering quality expected in industry and is a strong representation of my software design and development capabilities.

Reflection on the Enhancement Process

Enhancing this artifact allowed me to revisit an early project with a more mature engineering perspective. The original program was intentionally simplistic and contained several structural and security flaws, which I documented in detail during my Milestone One code review. Transforming that analysis into a fully enhanced artifact taught me several important lessons:

What I Learned

  • How to refactor legacy or reverse‑engineered code into a clean, object‑oriented architecture
  • How to eliminate global state and replace it with controlled class‑based logic
  • How to apply defensive programming techniques in C++
  • How to design software that is easier to maintain, extend, and test
  • How to document design decisions clearly for future developers

Challenges Faced

  • Untangling tightly coupled logic without altering the programs intended behavior
  • Designing a class structure that preserved functionality while improving clarity
  • Ensuring that permission logic, input validation, and state transitions were both secure and user‑friendly
  • Balancing readability with simplicity in a small, menu‑driven application

Overall, this enhancement strengthened my skills in software engineering, secure coding, and maintainable design. It also demonstrated my ability to take a flawed artifact, analyze it systematically, and transform it into a professional‑quality software component suitable for inclusion in my ePortfolio.
